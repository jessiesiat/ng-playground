2-way Data Binding

Angular data binding works in such a way that when a model changes the corresponding view template updates. Or someone manipulates the dom that changes the model, the view updates.

A good example is using filter to filter the data displayed in the view.

Our view will look something like this.

Sort by: <select ng-model="orderProp"> <option value="name">Alphabetical</option> <option value="age">Age</option> </select> <ul> <li ng-repeat="user in users | orderBy:orderProp"> <strong>{{ user.happy | expression }} {{user.name}}</strong> - <em>{{user.email}}</em> - {{user.age}} yrs. old </li> </ul>

And our controller, will default the orderProp to age.

ngPlayground.controller('UserCtrl', function ($scope) { $scope.users = [ {'name': 'John Doe', 'email': 'johndoe@example.com', 'happy': false, 'age': 38 }, {'name': 'Juan Dela Cruz', 'email': 'juandelacruz@example.com', 'happy': true, 'age': 25 } ]; $scope.orderProp = 'age'; });

We initially set the orderProp model to age, now when the page loads it will sort the list by Age thats how the model updates the view. By the time user change the dom selection to Alphabeticaly it will update the model and then the view to filter alphabeticaly. Thats is data binding!

Result: Sort by:
  • {{ user.happy | expression }} {{user.name}} - {{user.email}} - {{user.age}} yrs. old
