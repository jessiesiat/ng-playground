View & Templates

Angular view is a projection of the model through angular templates. That is when a model changes, Angular refreshes the binding points which will also updates the view.

<!doctype html> <html ng-app="ngPlayground"> <head> .. <script type="text/javascript" src="https://ajax.googleapis.com/ajax/libs/angularjs/1.2.1/angular.min.js"></script> <!-- will include js/controllers.js --> <script src="js/controllers.js"></script> </head> <body ng-controller="UserCtrl"> <ul> <li ng-repeat="user in users"> <strong>{{user.name}}</strong> - <em>{{ user.email }}</em> </li> </ul> </body> </html>

In here we are using ng-repeat attribute which is an Angular ngRepeat directive. This is an Angular repeater which tell's Angular to create lielement for each users in the list using the first li as the template.

We have also an Angular expression wrap in curly braces {{user.name}} and {{user.email}}, this denotes binding. This expressions are referring to our application model which is set up in our controller, in this case UserCtrl

Our js/controllers.js

var ngPlayground = angular.module('ngPlayground', []); ngPlayground.controller('UserCtrl', function ($scope) { $scope.users = [ {'name': 'John Doe', 'email': 'johndoe@example.com', 'happy': false, 'age': 38 }, {'name': 'Juan Dela Cruz', 'email': 'juandelacruz@example.com', 'happy': true, 'age': 25 } ]; });

We first create Angular module and name it ngPlayground as the name of our ngApp derective we have defined earlier. From it we have defined our controller, the first parameter is the name of the controller and the second is its definition. As you can see we passed in the $scope dependency, this is where we will attach our model attributes for it to be available to our view!

Result:
  • {{user.name}} - {{user.email}}
